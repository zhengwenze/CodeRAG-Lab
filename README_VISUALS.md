# CodeRAG Lab 视觉设计规范

- 目标：提升可读性、可维护性与专业感，确保架构信息以清晰、一致的方式呈现。
- 字体与排版：建议使用系统字体栈，避免过于花哨的网页字体，段落与标题层级清晰，正文 14-16px，标题 20-28px 递增。
- 颜色与对比：主文字使用深色系，背景保持清晰对比；强对比场景使用高亮颜色（如强调按钮、重要状态）。建议使用 CSS 变量来统一颜色值（在未来前端实现中可落地）。
- 架构图与图表：优先使用 Mermaid Diagram 进行可视化表达，避免图片体积过大或图片版权问题。
- 代码块与高亮：在需要展示代码时，使用 Markdown 代码块并开启语法高亮（前端实现可结合 Prism/Highlight.js），提升可读性。
- 响应式与移动端：确保主要信息在常见屏幕尺寸下可读，必要时提供简化视图。
- 交互与导航：清晰的导航结构、易用的 TOC、跳转链接，确保用户能快速定位内容。
